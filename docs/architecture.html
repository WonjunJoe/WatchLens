시스템 개요

WatchLens는 Google Takeout 데이터(YouTube 시청/검색 기록, Instagram 활동 기록)를 업로드하면, 시청 습관과 디지털 웰빙을 시각적으로 분석해주는 풀스택 웹 애플리케이션입니다.

유튜브가 제공하지 않는 인사이트 — 카테고리별 분석, Shorts 중독도, 도파민 지수, 시청 유형 분류(16가지), 콘텐츠 다양성, 폭식 세션, 검색→시청 전환율 등 — 을 19개 섹션으로 실시간 스트리밍합니다.

백엔드 아키텍처

백엔드는 Router → Service → Repository 3계층으로 분리되어 있습니다. Service 레이어의 모든 함수는 순수 함수(side-effect 없음)로, DB 호출 없이 데이터를 받아 계산 결과만 반환합니다.

API 엔드포인트

Upload Router Backend

POST /api/upload/youtube-takeoutZIP 업로드 (SSE)
POST /api/upload/watch-history시청 기록 JSON
POST /api/upload/search-history검색 기록 JSON

ZIP 내에서 한국어/영어 경로 모두 자동 탐지. 파서 → DB 저장 → YouTube API 메타데이터 수집 → Storage 백업까지 SSE로 진행률 스트리밍.

Stats Router Backend

GET /api/stats/period데이터 기간 조회
GET /api/stats/dashboard대시보드 생성 (SSE)
GET /api/stats/dashboard/cached캐시된 결과 반환

19개 섹션을 순차 계산하며 각각 SSE 이벤트로 전송. 완료 후 결과를 DB에 캐싱.

Instagram Router Backend

POST /api/instagram/uploadZIP 업로드 + 분석 (SSE)
GET /api/instagram/dashboard캐시된 결과 반환

Wellbeing Router Backend

GET /api/wellbeing/compute크로스플랫폼 웰빙 점수

YouTube + Instagram 데이터 모두 필요. 도파민·폭식·시청강도·러커·심야활동 5개 축으로 종합 점수 산출.

SSE 스트리밍 훅 — useSseStream

업로드와 대시보드 생성 모두 Server-Sent Events를 사용합니다. 커스텀 훅 useSseStream이 인증 토큰 주입, 이벤트 파싱, 취소(AbortController)를 처리합니다.

// 사용 예시
const { stream } = useSseStream();

await stream(url, ({ event, data }) => {
  if (event === "progress") setProgress(data);
  if (event === "section")  setSection(data.name, data.data);
  if (event === "done")     setLoading(false);
  if (event === "error")    setError(data.message);
});

SSE 이벤트 형식: event: section\ndata: {"name":"summary","data":{...},"loaded":1,"total":19}\n\n

Row-Level Security (RLS)

4개 테이블에 RLS 정책이 적용되어, 각 사용자는 자신의 데이터만 접근할 수 있습니다. 백엔드 쿼리의 user_id 필터와 이중으로 보호합니다 (defense-in-depth).

-- 예시: watch_records RLS
ALTER TABLE watch_records ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Users access own watch_records" ON watch_records
  FOR ALL USING (user_id = auth.uid()::text);

Backend — auth.py

def get_current_user(credentials):
    token = credentials.credentials
    res = supabase.auth.get_user(token)
    if not res.user:
        raise HTTPException(401)
    return res.user.id  # → user_id

모든 라우터에서 Depends(get_current_user)로 주입. PyJWT 대신 Supabase SDK로 직접 검증하여 JWT Secret이 불필요.

Frontend — AuthContext

// 로그인
supabase.auth.signInWithOAuth({
  provider: 'google',
  options: { redirectTo: '/upload' }
})

// 상태 동기화
supabase.auth.onAuthStateChange(
  (event, session) => {
    setUser(session?.user ?? null);
    setSession(session);
  }

보안 계층 (Defense-in-Depth)

계층메커니즘역할
1. 프론트엔드ProtectedRoute미인증 사용자 접근 차단
2. API 헤더Bearer Token 주입모든 API 호출에 인증 정보 포함
3. 백엔드Depends(get_current_user)토큰 검증 + user_id 추출
4. 쿼리 필터.eq("user_id", user_id)데이터 접근 범위 제한
5. DBRow-Level Security (RLS)DB 레벨 데이터 격리 (최종 방어선)

설계 패턴

SSE 스트리밍

업로드와 대시보드 생성 모두 WebSocket 대신 Server-Sent Events를 사용합니다. 단방향 스트리밍으로 충분한 유스케이스에서 구현 복잡도를 낮추면서 실시간 진행률을 제공합니다.

Context 기반 상태 관리

Redux 없이 React Context API로 전역 상태를 관리합니다. SSE 이벤트가 도착할 때마다 setSection()으로 점진적 업데이트하여 컴포넌트가 데이터가 도착하는 즉시 렌더링됩니다.

Repository 패턴

모든 DB 쿼리가 repository.py 한 곳에 집중되어 있습니다. Service 레이어는 DB를 직접 호출하지 않고, 데이터를 받아 순수 계산만 수행합니다. 테스트와 DB 교체가 용이합니다.

순수 함수 계산 레이어

stats_service.py의 17개 compute_* 함수와 instagram_stats.py의 14개 compute_ig_* 함수는 모두 side-effect가 없는 순수 함수입니다. 입력 데이터만으로 결과를 계산합니다.

캐싱 전략

대시보드 결과를 JSONB 컬럼에 캐싱하여 재방문 시 즉시 로드합니다. 새 파일 업로드 시 캐시를 자동 무효화하여 stale 데이터를 방지합니다.

중앙화된 설정

모든 임계값, 가중치, 매핑 테이블이 config/settings.py에 집중되어 있어 핵심 로직 수정 없이 파라미터만 조정할 수 있습니다.
